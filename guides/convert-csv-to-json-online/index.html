What the conversion looks like

This CSV:

name,city,plan
Ada,"Austin, TX",pro
Grace,London,free

becomes this JSON:

[
  { "name": "Ada",   "city": "Austin, TX", "plan": "pro" },
  { "name": "Grace", "city": "London",     "plan": "free" }
]

Note the second field of the first row: "Austin, TX" contains a comma, so it's wrapped in quotes in CSV. A converter that just splits lines on commas would break that row into four columns — this is the single most common way CSV conversions silently corrupt data.

Why quoting rules matter

  • Embedded commas — "Austin, TX" must stay one value.
  • Embedded quotes — CSV escapes a quote by doubling it: "She said ""hi""".
  • Embedded newlines — a quoted value can span lines; splitting the file on newlines first mangles it.

These rules (RFC 4180) are exactly what spreadsheet apps emit when you export, so a converter must speak them to round-trip real-world files.

Common uses

  • Spreadsheet → API: export a sheet as CSV, convert, and you have the JSON payload your endpoint expects.
  • API → spreadsheet: paste a JSON response, convert to CSV, open in Excel or Google Sheets for the non-developers on your team.
  • Quick inspection: JSON's structure makes a messy export readable; CSV makes a JSON dump sortable and filterable.
